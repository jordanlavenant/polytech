# Chapitre 2 : Structures linéaires

## 1. Tableaux dynamiques

Un tableau est une structure de données contiguë en mémoire (un seul gros bloc), qui permet l'accès aux valeur qu'il contient par indexage.

Pour un tableau $t$ de taille $n$ ses valeurs sont : $t[0], t[1], ..., t[n]$

Sa taille est **fixe**, détermiéne lors de sa création. Pour "agrandir" un tableau, il faut en créer un nouveau, plus grand dans lequel on **recopie** les valeurs de l'ancien.

Pour stocker un nb de variable de valeurs dans un tableau, on ne remplit que le début des cases du tableau, en laissant une valeur "vide" pour les suivantes.

_Exemple: Null (C), None (Python), ..._

### 1. Opérations élémentaires

- **Accès** (lecture / écriture), à l'élément d'indice $i$ dans $t$. $\rightarrow$ coût $\theta(1)$

- **Ajout** d'un nouvel élément en position $i$ (Si $\le n-1$ valeurs déjà présentes dans $t$)

```py
[1, 2, 3, None, None, None]
```

Si on insère en l'indice $i$, il faudra décaller $n-i$ valeurs (à droite de l'indice i) $\rightarrow \theta(n-i)$

- **Suppression** de l'élément en position $i$. Nécessité de décaller $n-i-1$ valeurs $\rightarrow -(n-i)$

### 2. Coût des réallocations en mémoire

On cherche maintenant à pouvoir faire un nombre arbitraire d'insertions dans le tableau $\rightarrow$ passe de tableau "statique", à tableau "dynamique". Cela implique un certain nombre de réallocations mémoire.

#### Algorithme

    t <- tableau vide() de taille K
    Pour i de 1 à n:
        x <- input()
        ajouter x à la fin de t
        Si t est saturé:
            Reallouer(t)

#### Option 1

$Reallouer(t:tableau)$

    n <- taille(t)
    t1 <- tableau de taille n+K
    Pout x dans t:
        Ajouter x à la fin de t1
    t <- t1

Avec $K$ à déterminer (entier fixe)

**Complexité :** $\theta(n)$

Les tailles successives prises par $t$ après chaque appel à $Reallouer(t)$ sont :

$$K, 2K, 3K, 4K, ...$$

Donc le j-ième appel à $Reallouer(t)$ est de coût $\theta(j.K)$

Leur coût total est donc

$$\sum_j^{n/K} \theta(j.K) = \theta (K \sum_j^{n/K} j)$$

$$= \theta(K. (\frac{n}{K})^2) = \theta(\frac{n^2}{K})$$

$=>$ Si $K$ est une constante indépendante de $\N$, le coût est **quadratique**.

#### Option 2

$Reallouer(t:tableau)$

    N <- taille(t)
    t1 <- tableau de taille 2N
    Pour x dans t:
        Ajouter x à la fin de t1
    t <- t1

Taille initiale de $t$ : $1$

Les tailles successives sont :

$$1, 2, 4, 8, 16, ..., 2^j$$

Le coût du j-ème appel à $Reallouer(t)$ est $\theta(2^j)$

Le dernier appel à $Reallouer$ doit retourner un tableau de taille $\ge n$

En équation, cela donne $2^j \ge n \Longleftrightarrow log_2(n)$

Le plus petit $j$ vérifiant cela est $j=log_2(n)$

Le coût total des reallocations est donc :

$$\sum^{log_2(n)}_{j=1} \theta(2^j) = \theta(\sum^{log_2(n)}_{j=1} 2^j) = \theta(n)$$

Le coût de $n$ insertions en partant d'un tableau déjà vide, avec l'**option 2**, est $\theta(n)$.

$\rightarrow$ On dit que le **coût amorti** de l'insertion est $\theta(1)$

## 2. Listes (doublement) chaînées

### 1. Définition

Une liste chaînée est une structure de données composées de cases stockées de façon disparate en mémoire, et reliées entre elles par un chemin de pointeurs. La liste est donnée par un pointeur vers la première case.

```
Structure Cellule
    valeur : type interne de la liste
    suiv : pointeur vers une Cellule
```

```
Structure Liste:
    debut : pointeur vers une Cellule
```

```
fonction taille(L:Liste)
    n <- 0
    C <- L . debut
    Tant que c != vide:
        n <- n + 1
        c <- c.suiv
    retourner n
```

### 2. Opérations élémentaires

- **Accès** (lecture / écriture) à l'élément en position $i$ : $\theta(i)$

- **Ajouter / SUpprimer** une case en position $i$ :

$\theta(1)$ si on connaît l'adresse de la case $i-1$.

$\theta(i)$ sinon (coût d'accès à la position $i-1 = \theta(1)$)

```
fonction supprimer(L:Liste, i: entree)
    c <- accesCase(L, i-1)
    res <- c.suiv
    Si c.suiv != vide:
        c.suiv <- c.suiv.suiv
    retourner res
```

### 3. Listes doublement chaînées

On part de la struc ture de liste (simplement), chainées, et on ajoute des pointeurs vers les cases précédentes dans la liste.

```
Structure Cellule
    valeur : type interne de la liste
    suiv : pointeur vers une Cellule
    prec : pointeur vers une Cellule
```

```
Structure Liste:
    debut : pointeur vers une Cellule
    fin : pointeur vers une Cellule
```

L'intérêt est de pouvoir supprimer en temps $\theta(1)$ une case dont on connait l'adresse dans une liste doublement chainée.

Incrémentation binaire

```
fonction incr():
    c <- cpt.debut
    Tant que c.suiv non-vide:
        Si c.valeur = 0:
            c.valeur <- 1
            retourner
        Sinon:
            c.valeur <- 0
            Si c.suiv vide:
                c.suiv <- Cellule(1)
                retourner
        c <- c.suiv
    retourner
```
